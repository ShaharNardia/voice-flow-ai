# QA/Staging Environment Specification

## Infrastructure

- **Firebase Project**: `voiceflow-ai-staging`
  - Enable Firestore (native mode), Authentication (Email/Password + Google), Storage, Functions.
  - Create service accounts for CI deployment (deploy, seed).
- **Hosting**: Firebase Hosting target `staging`.
- **Emulator Support**: Configure Firebase Emulator Suite for local smoke runs (FireStore/Functions/Auth).

## Configuration Artifacts

| File | Description | Location |
| --- | --- | --- |
| `lib/firebase_options_staging.dart` | Flutter options generated by `flutterfire configure --project=voiceflow-ai-staging` | Repo |
| `web/manifest-staging.json` | Web manifest with staging icons/URLs | Repo |
| `firebase/.env.staging` | Secrets (Stripe keys, Twilio, VAPI) | Git-ignored |
| `firebase/staging-service-account.json` | CI deploy credentials | Secret manager |

Ensure CI injects environment variables securely (GitHub Actions secrets or Azure Key Vault). Never commit raw secrets.

## Deployment Pipeline

1. **Branching**: use `release/*` branches for staging.
2. **CI workflow**:
   - `flutter build web --dart-define-from-file=env/staging.json`.
   - `firebase deploy --only hosting:staging,functions` with staging service account.
   - Run smoke automation suite post-deploy (see `docs/testing/automation/smoke_suite.md`).

## Seed Data

Create `scripts/seed_staging.dart` (Dart CLI) that writes deterministic data:

| Collection | Records |
| --- | --- |
| `Users` | Admin, Dispatcher, Technician demo accounts |
| `Jobs` | 5 open jobs, 2 completed |
| `Leads` | 5 new leads |
| `Invoices` | 3 invoices referencing Stripe test customers |

Execution:
```bash
dart run scripts/seed_staging.dart --project voiceflow-ai-staging
```

Reset schedule: nightly via GitHub Action or Firebase scheduled function.

## Access Control

- Grant read-only access to QA team; write access to automation service account.
- Require MFA for admin accounts.
- Create Firebase Auth test users with naming convention `qa_<role>@example.com`.

## Monitoring & Logging

- Enable Crashlytics and Performance Monitoring for staging.
- Ship logs to BigQuery dataset `staging_logs` for replay/analysis.
